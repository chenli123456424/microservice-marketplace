## 问题理解
你说得对！订单商品图片应该**实时从 `product_image` 表获取**，而不是存储在 `order_item` 表中。

这样做的好处：
- ✅ 商品图片更新后，订单中显示的也是最新图片
- ✅ 不需要在 `order_item` 表中冗余存储图片路径
- ✅ 数据库更规范，遵循关联查询的设计原则

## 已完成的修复

### 1. 修改 OrderItemMapper

**文件**: `user-service/src/main/java/com/example/microservice/user/mapper/OrderItemMapper.java`

**修改前**:
```java
@Select("SELECT * FROM `order_item` WHERE order_id = #{orderId}")

```

**修改后**:
```java
@Select("SELECT oi.*, " +
        "(SELECT pi.image_url FROM product_image pi WHERE pi.product_id = oi.product_id ORDER BY pi.sort LIMIT 1) as productImage " +
        "FROM order_item oi WHERE oi.order_id = #{orderId}")
List<OrderItem> selectByOrderId(Long orderId);
```

**说明**: 
- 查询订单商品时，通过子查询从 `product_image` 表实时获取图片
- 参考了 `CartItemMapper` 的设计模式
- `ORDER BY pi.sort LIMIT 1` 获取第一张图片（主图）

## 技术原理

### 查询流程

1. **前端请求订单列表**: `GET /api/orders/list`

2. **后端查询订单**: 
   ```java
   List<Order> orders = orderMapper.selectOrdersByUserId(userId, ...);
   ```

3. **后端查询订单商品**（关键！）:
   ```java
   List<OrderItem> orderItems = orderItemMapper.selectByOrderId(orderId);
   ```

4. **SQL执行**（自动关联商品图片）:
   ```sql
   SELECT oi.*, 
          (SELECT pi.image_url 
           FROM product_image pi 
           WHERE pi.product_id = oi.product_id 
           ORDER BY pi.sort 
           LIMIT 1) as productImage
   FROM order_item oi 
   WHERE oi.order_id = 1
   ```

5. **返回给前端**:
   ```json
   {
     "orderItems": [
       {
         "productId": 3,
         "productName": "坚朗 断桥铝门窗五金件",
         "productImage": "/uploads/0fda3c6c-0459-46f8-80e2-49f5e45bb0a5.png",
         ...
       }
     ]
   }
   ```

6. **前端显示图片**:
   ```javascript
   <img src={`http://localhost:8081${item.productImage}`} />
   ```
   最终URL: `http://localhost:8081/uploads/0fda3c6c-0459-46f8-80e2-49f5e45bb0a5.png`
